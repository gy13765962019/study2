- **Hotspot 修复（08:54）**：
  - 现象：用户截图显示 3 个热点渲染为黑色方块
  - 根因：`data:image/svg+xml,xxx` URL 在 A-Frame 1.4.2 + GitHub Pages 下被 `crossorigin` 策略或 `look-controls` 上下文拒绝渲染；纯 SVG 缺纹理 fallback 时表现为黑色 plane
  - 修复：
    1. 把热点底图改为真实 SVG 文件 `assets/hotspot/forward.svg`（256×256，渐变光环+发光箭头+中心白点）
    2. `<a-assets>` 增加 `<img id="hotspot-img" src="assets/hotspot/forward.svg">` 预加载
    3. 去掉所有 `crossorigin="anonymous"`（同源不需要）
    4. 热点结构：`<a-entity>` 容器 → 3 子元素（中心箭头 + 双层扩散环），全部 `look-at="[camera]"` 永远朝相机
    5. 内置 `<a-animation>` 驱动：箭头 1.4s 呼吸缩放 0.9↔1.15、双环 1.8s 交替缩放 0.5↔1.8 + 透明度 0.9→0（径向扩散错峰 0.6s）
    6. 额外：每个热点上方挂一个 `<a-plane>`+`<a-text>` 悬浮标签，显示目标场景名（"博学苑"等）
  - 验证：HEAD `https://gy13765962019.github.io/study2/assets/hotspot/forward.svg` → 200 / `image/svg+xml` / 1077 bytes；index.html curl 拉取确认含 `forward.svg`、`look-at`、`a-animation` 等新代码
